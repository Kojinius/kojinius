1. 概要

Typolish（タイポリッシュ）は、画像・PDF・Web・動画・Office ファイルをアップロードしてピンポイントのフィードバック（アノテーション）と承認ワークフローを管理するビジュアル校正・プルーフィングツールです。共有リンクで非ログインゲストもアノテーション・承認に参加できます。

語源は Typo（誤字）+ Polish（磨く）。ドメインは typolish.com（さくらインターネット取得）。

現在は Vercel 本番稼働中です（A レコード: 76.76.21.21）。Firebase App Hosting から Vercel への移行（DNS 切替）は 2026-05-24 に完了しており、apphosting.yaml は削除済みです。

料金プラン

Free ¥0 / Pro ¥4,980 / Business ¥14,800（月額・税込表示は契約書に準じます）。プロジェクト単位課金でメンバーは無制限招待できます。1 ライセンス = 1 アカウントで、アカウント共有・同時ログインは単一セッション強制により制御されています。

Service Account JSON の投入方法

FIREBASE_SERVICE_ACCOUNT と GOOGLE_APPLICATION_CREDENTIALS_JSON は GCP で発行した Service Account JSON の内容を 1 行化（改行を削除）して Vercel env に投入します。Vercel では ADC（Application Default Credentials）が使えないため、JSON の直接設定が必要です。

gcloud iam service-accounts keys create key.json --iam-account=<SA email>
cat key.json | tr -d '\n'   # 1 行化して Vercel env に貼り付け

6. Cron Jobs（5本）

vercel.ts の crons に定義された 5 本です。Production 環境のみ実行され、Preview 環境では実行されません。

Vercel が自動で Authorization: Bearer <CRON_SECRET> ヘッダーを付与するため、各エンドポイントはこのヘッダーで認証を行います。

エンドポイントスケジュールJST役割
/api/cron/expiry-notifications 0 0 * * * 毎日 9:00 プルーフ期限が近いユーザーへの通知メール送信
/api/cron/presence-sweep */2 * * * * 2 分ごと RTDB のプレゼンス（在席情報）の stale エントリ掃除
/api/cron/expire-free-proofs 0 18 * * * 毎日 3:00 Free プランで 30 日経過した proof に deletionMarker をセット（論理削除）。R2 オブジェクトはこの時点では削除しません
/api/cron/hard-delete-expired-proofs 0 18 * * * 毎日 3:00 論理削除（deletionMarker）から 7 日経過した proof の物理削除（R2 オブジェクト含む）。冪等設計のため expire-free-proofs との並列実行に対応しています
/api/cron/notify-pat-expiry 0 18 * * * 毎日 3:00 PAT（Personal Access Token）の期限 7 日前・1 日前に通知メール送信。期限切れ PAT を自動失効（disabledAt セット）
Cron の動作確認方法

Vercel Dashboard > プロジェクト > Crons タブで各 Cron の実行ログと最終実行時刻をご確認いただけます。手動発火は Authorization: Bearer <CRON_SECRET> ヘッダーを付けて当該エンドポイントを curl することで可能です。

Vercel プランについて

最長 540 秒の Function は Vercel Pro プラン以上で動作します。Hobby プランは 300 秒固定上限のため、/api/v1/integrations/proofs/process（外部連携 Proof 転送）がタイムアウトします。Vercel Pro プランの maxDuration 上限は 800 秒で、540 秒は余裕を持って収まります。

8. 外部連携

① MCP サーバー（/api/mcp）

PAT（Personal Access Token）認証で外部からアクセスできる MCP サーバーです。Typolish の各種操作（proof 作成・アノテーション・承認等）を外部ツールから呼び出せます。PAT は /developers ページで発行・管理します。

② Integration API（/api/v1/integrations/* — 15 ルート）

HMAC 署名認証の外部連携 API です。現在 Kalenex（橋本晃治保有）が接続しています。

  • HMAC 検証: src/lib/integration-auth.ts
  • Webhook 配信: src/lib/integration-webhook.ts
  • Integration 設定: Firestore integrations/{clientId}（clientSecret / webhookSecret / allowedScopes / webhookUrl）
  • 自己管理 UI: /developers ページ（プロジェクト Owner が管理）
Kalenex との連携について

Kalenex は橋本晃治保有の別サービスで、譲渡対象外です。Kalenex との Integration API 連携を継続するか停止するかは、譲渡契約で明確にしていただけますようお願いします。継続の場合、integrations/{clientId} の clientSecret を rotate することをお勧めします。

③ Chrome 拡張機能 v2.2.0

Chrome Web Store 公開済みの「Typolish - Page Capture」拡張機能です。typolish.com 上で content-script として動作し、Web ページキャプチャを Typolish に送信します。

  • 認証: /api/extension/issue-token でトークン取得
  • コールバック: /api/extension/render-callback へ POST
  • Chrome Web Store Item オーナーも譲渡対象です（Publisher Group に橋本晃治を追加することで保守継続も可能です）

13. 権限マトリクス（B案）

B案（運用 = SKYDIRECTION 様・保守 = 橋本晃治）を前提とした権限設計です。各リソースで Owner（運用権）と Maintainer（保守権）を分離しています。

権限移行の順序として 「Owner 追加 → 旧 Owner 降格」 の順で実行することで、一時的なアクセス不能を防ぐことができます。
